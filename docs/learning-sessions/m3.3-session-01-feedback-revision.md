# M3.3 学习复盘 01：审核意见回流修订与 reject 终止约束

## 1. 本次实现了什么
- 增加结构化回流 payload：支持候选增删、主题词/搜索词、来源启停/权重、排序权重、editor notes。
- 增加 `feedback executor`：在已有 review snapshot 基础上执行修订，重排并回到 `final_review`。
- 增加 runtime config 持久化：回流中的来源与排序调整写入 `outputs/runtime-config/global.json`，后续 run 自动生效。
- 增加 reject 终止语义：当前 run 被 reject 后，`recheck/watchdog` 均不会发布该 run。
- 增加 run 级指令时间边界：只读取 `reviewStartedAt` 之后的审核动作，避免历史 reject 污染新 run。

## 2. 流程图（M3.3）
```text
request_revision (Feishu/CLI instruction)
  -> instruction-store 读取 latest action（decidedAt >= reviewStartedAt）
  -> feedback-schema 归一化并校验
  -> feedback-executor 执行：
       - candidate add/remove
       - runtime config merge
       - rerank + rebuild metrics/highlights/outline
  -> review_stage = final_review（等待终稿审核）

reject
  -> review_status = rejected
  -> publish_or_wait 强制 shouldPublish=false
  -> watchdog/recheck 跳过发布
```
